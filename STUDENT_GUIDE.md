## 3. ระบบ XP และ Rank

### 3.1 XP คืออะไร?

**XP (Experience Points)** คือคะแนนสะสมที่ได้จากการทำกิจกรรมต่างๆ ในระบบ เมื่อสะสม XP ถึงเกณฑ์ จะ **ขึ้น Rank** อัตโนมัติ

### 3.2 วิธีได้รับ XP

| กิจกรรม | XP ที่ได้ | เพิ่มเติม |
|---|---|---|
| ส่งงานได้ 100% | **+50 XP** | +10 🪙 |
| ส่งงานได้ 80-99% | **+30 XP** | +5 🪙 |
| ส่งงานได้ 50-79% | **+15 XP** | +2 🪙 |
| ส่งงานได้ <50% | **+5 XP** | — |
| **ผ่านโจทย์ครั้งแรก** (First Solve) | **+20 XP** เพิ่มเติม | +5 🪙 +1 💎 |
| Login ทุกวัน (Streak) | **+10 XP** | +2 🪙 |
| Streak 3 วันติด | **+30 XP** รวม | +7 🪙 |
| Streak 7 วันติด | **+60 XP** รวม | +12 🪙 +2 💎 |
| เล่น Quiz Blitz (ครั้งแรกของวัน) | **+25 XP** | +10 🪙 |
| เล่น Bug Hunt (ครั้งแรกของวัน) | **+30 XP** | +12 🪙 |
| เล่น Code Autopsy (ครั้งแรกของวัน) | **+20 XP** | +8 🪙 |
| ได้รับ Achievement | +50 ถึง +1,000 XP | ขึ้นอยู่กับ Achievement |

> 💡 **เคล็ดลับ:** ถ้ามี Season ที่ Active อยู่ XP ทุกอย่างจะถูกคูณด้วย Multiplier เช่น x2!

## 4. เรียนและทำโจทย์

### 4.1 เข้าดูโจทย์

1. กดเมนู **"รายวิชา"** → เลือกวิชา
2. เลือกโจทย์ที่ต้องการทำ

---

📸 **[แคปเจอหน้าจอ: หน้ารายการโจทย์ — แสดงโจทย์แต่ละข้อพร้อม Tag ระดับความยาก (Easy/Medium/Hard) และสถานะผ่าน/ไม่ผ่าน]**

---

### 4.2 หน้าเขียนโค้ด (Coding Workspace)

หน้านี้แบ่งเป็น 2 ฝั่ง:

**ฝั่งซ้าย — โจทย์:**
- ชื่อโจทย์ และคำอธิบาย
- ตัวอย่าง Input/Output
- ผลการทดสอบ (Test Cases)

**ฝั่งขวา — เขียนโค้ด:**
- Editor พร้อม Syntax Highlight ภาษา C
- มีโค้ดตั้งต้นให้แล้ว (Starter Code)

---

📸 **[แคปเจอหน้าจอ: Coding Workspace — แสดง Layout 2 ฝั่ง โจทย์ซ้าย Editor ขวา]**

---

### 4.3 เขียนและส่งโค้ด

1. เขียนโค้ด C ใน Editor ฝั่งขวา
2. กดปุ่ม **"▶ Run"** เพื่อทดสอบ
3. ดูผลแต่ละ Test Case (✅ ผ่าน / ❌ ไม่ผ่าน)
4. แก้ไขจนผ่านทุก Test Case
5. ระบบให้ XP อัตโนมัติทันที

---

📸 **[แคปเจอหน้าจอ: ผลการ Run — แสดง Test Case ที่ผ่าน (✅) และไม่ผ่าน (❌) พร้อม Output จริงกับ Output ที่คาดหวัง]**

---

### 4.4 รับ XP หลังส่งงาน

เมื่อส่งงานสำเร็จ ระบบแสดง Toast แจ้ง:

```
🎉 ได้รับ +50 XP, +10 CodeCoin!
✨ First Solve Bonus! +20 XP เพิ่มเติม
```

---

📸 **[แคปเจอหน้าจอ: Toast แจ้ง XP — แสดงข้อความ "+50 XP" หลังส่งงานผ่าน]**

---

### 4.5 เคล็ดลับการเขียนโค้ด C

```c
// โครงสร้างพื้นฐาน
#include <stdio.h>

int main() {
    // เขียนโค้ดที่นี่
    printf("Hello, World!\n");
    return 0;
}
```

| สิ่งที่ต้องจำ | ตัวอย่าง |
|---|---|
| ทุกคำสั่งต้องจบด้วย `;` | `int x = 5;` |
| รับค่าจาก Input ด้วย `scanf` | `scanf("%d", &x);` |
| พิมพ์ผลด้วย `printf` | `printf("%d\n", x);` |
| ต้องมี `return 0;` ท้าย main | `return 0;` |

## 5. การขอ Hint จาก AI Coach

ถ้าติดปัญหาและคิดไม่ออก สามารถขอ Hint จาก AI ได้

### 5.1 วิธีขอ Hint

1. อยู่ในหน้าเขียนโค้ด กดปุ่ม **"💡 ขอ Hint"**
2. AI Coach จะให้ Hint แบบ **Socratic** (ไม่บอกคำตอบตรงๆ แต่ชี้แนวทาง)

---

📸 **[แคปเจอหน้าจอ: ปุ่ม "💡 ขอ Hint" และ Hint Level 1 ที่ AI ให้ — แสดงคำถามชวนคิด]**

---

### 5.2 Hint มี 4 ระดับ

| ระดับ | สิ่งที่ได้รับ |
|---|---|
| Hint 1 | คำถามชวนคิด (Socratic) — ไม่บอกคำตอบ ช่วยให้คิดเอง |
| Hint 2 | อธิบาย Concept + ตัวอย่างคล้ายกันแต่ไม่ใช่โจทย์นั้น |
| Hint 3 | โครงสร้าง Pseudocode / Algorithm ทีละขั้น |
| Hint 4 | วิเคราะห์โค้ดที่เขียน — บอกว่า Logic ผิดตรงไหน |

> 💡 ลองคิดเองก่อนขอ Hint ระดับ 4 เสมอ จะได้เรียนรู้มากกว่า!

### 5.3 Mindset Coach (เมื่อล้มเหลวบ่อย)

ถ้าส่งงานผิด **3 ครั้งติดต่อกัน** ระบบจะแสดงข้อความให้กำลังใจจาก AI Coach อัตโนมัติ

### 5.4 Challenge Coach (เมื่อทำได้ดี)

ถ้าส่งงานได้ **90% ขึ้นไป** ระบบจะแนะนำโจทย์ที่ยากขึ้นและ Concept ที่ควรเรียนต่อ

### 5.5 Predictive Risk Alert

ถ้าคะแนนมีแนวโน้มลดลงหรือล้มเหลวซ้ำ ระบบจะแจ้งเตือนอัตโนมัติพร้อมแนะนำวิธีแก้ไข

---

📸 **[แคปเจอหน้าจอ: Mindset Coach Message — แสดงข้อความให้กำลังใจหลังล้มเหลว 3 ครั้ง]**

## 7. Mini-Games

---

### 7.2 Game 1 — Quiz Blitz 🧩 (ควิซซ์จับเวลา)

**วิธีเล่น:**
1. กดปุ่ม **"เล่น Quiz Blitz"**
2. เลือกหน่วยเนื้อหา
3. ตอบคำถาม Multiple Choice 5 ข้อ (30 วินาที/ข้อ)
4. ดูคะแนนและ XP ที่ได้

**ตัวอย่างคำถาม:**
```
ข้อ 1: คำสั่งใดใช้รับค่าตัวเลขจำนวนเต็มใน C?
  A) printf("%d", x)
  B) scanf("%d", &x)  ← ถูก
  C) input(x)
  D) read(x)
```

---

📸 **[แคปเจอหน้าจอ: Quiz Blitz ระหว่างเล่น — แสดงข้อคำถาม, 4 ตัวเลือก, Timer countdown และคะแนน]**

---

📸 **[แคปเจอหน้าจอ: Quiz Blitz หน้าสรุปผล — แสดงคะแนน X/5, XP ที่ได้, และปุ่มเล่นอีกครั้ง]**

---

### 7.3 Game 2 — Code Autopsy 🔬 (วิเคราะห์โค้ด)

**วิธีเล่น:**
1. กดปุ่ม **"เล่น Code Autopsy"**
2. อ่านโค้ด C ที่แสดง
3. เลือก Output ที่โปรแกรมจะพิมพ์ (4 ตัวเลือก)
4. ดูคำอธิบายหลังตอบแต่ละข้อ

**ตัวอย่าง:**
```c
int x = 5;
printf("%d", x * 2);
// Output คืออะไร?  →  10
```

---

📸 **[แคปเจอหน้าจอ: Code Autopsy ระหว่างเล่น — แสดงโค้ด C และ 4 ตัวเลือก Output]**

---

### 7.4 Game 3 — Bug Hunt 🐛 (ตามล่าบัค)

**วิธีเล่น:**
1. กดปุ่ม **"เล่น Bug Hunt"**
2. อ่านโค้ด C ที่มี **Bug** (ข้อผิดพลาด)
3. พิมพ์วิธีแก้ Bug ลงในช่อง
4. AI ตรวจคำตอบ (ไม่ต้องพิมพ์ถูก 100% ตัวต่อตัว)

**ตัวอย่าง Bug:**
```c
// โค้ดมีบัค:
int i;
for (i = 0; i < 5; i++)
    printf("%d ", i)   // ← ขาด semicolon!

// คำตอบ: เพิ่ม ; ท้ายบรรทัด printf
```

---

📸 **[แคปเจอหน้าจอ: Bug Hunt ระหว่างเล่น — แสดงโค้ดที่มี Bug highlight สีแดง และช่องพิมพ์คำตอบ]**

## 12. Free Editor

### 12.1 เข้าใช้งาน

กดเมนู **"เครื่องมือ"** → **"✏️ Free Editor"** หรือไปที่ URL `#/student/editor`

### 12.2 รายละเอียด

- เขียนโค้ดและรันได้โดยไม่ต้องมีโจทย์
- เลือก **Theme** ได้หลาย Style (Dracula, Monokai, One Dark ฯลฯ)
- เลือก **Font** ได้ (JetBrains Mono, Fira Code ฯลฯ)
- มีเครื่องมือ **Drawing/Annotation** — วาดบนหน้าจอได้
- เหมาะสำหรับทดลองเขียนโค้ด ก่อนนำไปใช้ในโจทย์จริง

## เคล็ดลับการสะสม XP เร็ว 🚀

1. **Login ทุกวัน** — ไม่ให้ Streak ขาด รับ Bonus สูงสุดวันละ +60 XP
2. **เล่นเกมทุกวัน** — 3 เกม × ~25 XP = +75 XP/วันจากเกมอย่างเดียว
3. **ผ่านโจทย์ครั้งแรก** — ได้ First Solve Bonus +20 XP ทุกโจทย์
4. **ตั้งใจส่งงาน** — Score 100% ได้ 10 เท่าของ Score <50%
5. **ไม่ขอ Hint เยอะ** — โจทย์ Hard ไม่ขอ Hint = No Hint Hero +200 XP

## คำถามที่พบบ่อย

**Q: ลืม Password ทำอย่างไร?**
A: กด "ลืมรหัสผ่าน" บนหน้า Login → ระบบส่ง Email Reset ให้

**Q: Leaderboard ทำไมมีชื่อน้อยคน?**
A: แสดงเฉพาะเพื่อนในวิชาเดียวกัน ไม่ใช่ทุกคนในโรงเรียน

**Q: ส่งงานแล้ว XP ไม่ขึ้น?**
A: ตรวจสอบว่า Login อยู่หรือเปล่า / Refresh หน้าแล้วลองใหม่

**Q: Streak หายทำอย่างไร?**
A: Streak รีเซ็ตถ้าไม่ได้ Login 1 วัน — ต้องเริ่มนับใหม่ ควร Login ทุกวัน

**Q: เล่นเกมซ้ำในวันเดียวกันได้มั้ย?**
A: ได้ แต่ได้ XP น้อยกว่า (Repeat Rate) ควรเล่นทุกเกมครั้งแรกของวันก่อน
