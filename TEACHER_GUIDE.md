## 3. จัดการรายวิชา

### 3.1 ไปที่หน้าจัดการรายวิชา

กดเมนู **"จัดการรายวิชา"** ใน Navbar หรือกดปุ่ม **"แก้ไข"** บน Course Card

---

📸 **[แคปเจอหน้าจอ: หน้าจัดการรายวิชา — แสดง form สร้างวิชา และรายการวิชาที่มีอยู่]**

---

### 3.2 สร้างวิชาใหม่

กรอกข้อมูล:

| ช่อง | ตัวอย่าง | หมายเหตุ |
|---|---|---|
| ชื่อวิชา | ว31281 การเขียนโปรแกรม ม.4/6 เทอม 1/2568 | ควรระบุห้อง/เทอมให้ชัดเจน |

| ภาษาโปรแกรม | C | เลือกจาก Dropdown |

จากนั้นกด **"สร้างวิชา"**

> ระบบสร้าง **Class Code** อัตโนมัติ นักเรียนใช้ Code นี้ลงทะเบียน

---

📸 **[แคปเจอหน้าจอ: Form สร้างวิชาใหม่ — ชี้ไปที่ช่อง Class Code ที่ระบบสร้างให้]**

---

### 3.3 เปิด/ซ่อนวิชา

- **เปิดสอน (Published):** นักเรียนเห็นวิชาและทำโจทย์ได้
- **ซ่อน (Hidden):** นักเรียนไม่เห็น (ใช้ระหว่างเตรียมเนื้อหา)

กดปุ่ม **"ซ่อน"** หรือ **"เปิดสอน"** บน Course Card ได้เลย

### 4.2 สร้างโจทย์ใหม่

| ช่อง | คำอธิบาย | ตัวอย่าง |
|---|---|---|
| ชื่อโจทย์ | หัวข้อโจทย์ | "ตัวแปรและการรับค่า" |

📸 **[แคปเจอหน้าจอ: Form สร้างโจทย์ใหม่ — แสดงช่องกรอกทุกช่อง โดยเฉพาะช่องระดับความยากและ Starter Code]**

## 5. ออก Test Cases

Test Case คือชุดทดสอบที่ระบบใช้ตรวจโค้ดนักเรียนอัตโนมัติ

### 5.1 เข้าหน้า Test Case Editor

กดปุ่ม **"Test Cases"** ข้างโจทย์ที่ต้องการ

หรือไปที่ Menu **"จัดการรายวิชา"** → เลือกโจทย์ → **"Test Cases"**

---

📸 **[แคปเจอหน้าจอ: หน้า Test Case Editor — แสดงรายการ Test Cases และ Form เพิ่ม Test Case ใหม่]**

---

### 5.2 สร้าง Test Case

| ช่อง | คำอธิบาย | ตัวอย่าง |
|---|---|---|
| Input | ข้อมูลที่ป้อนให้โปรแกรม | `Assawin` |
| Expected Output | ผลลัพธ์ที่โปรแกรมต้องพิมพ์ | `Hello, Assawin` |
| คะแนน | คะแนนของ Test Case นี้ | 25 |
| Hidden | ซ่อนจากนักเรียน | ✓ (สำหรับ Test Case ลับ) |

> **เคล็ดลับ:** ควรมีทั้ง Test Case ที่นักเรียนเห็น (สาธารณะ) และซ่อน (Secret) เพื่อป้องกันการ Hardcode คำตอบ

---

📸 **[แคปเจอหน้าจอ: Form เพิ่ม Test Case — ชี้ให้เห็น checkbox "Hidden" และช่อง Input/Output]**

---

### 5.3 ตัวอย่าง Test Case สำหรับโจทย์ Hello World

```
Test Case 1 (สาธารณะ):
  Input:    Somchai
  Output:   Hello, Somchai
  คะแนน:   50

Test Case 2 (ซ่อน):
  Input:    (ว่าง)
  Output:   Hello, World
  คะแนน:   50
```

## 7. วิเคราะห์ผลการเรียน

กดเมนู **"วิเคราะห์นักเรียน"** ใน Navbar

### 7.1 เลือกรายวิชา

ด้านบนสุดมี Dropdown **"เลือกรายวิชา"** — เลือกวิชาที่ต้องการดูข้อมูล

> ข้อมูลทั้ง 7 แท็บจะเปลี่ยนตามวิชาที่เลือกอัตโนมัติ

---

📸 **[แคปเจอหน้าจอ: หน้า Analytics — ชี้ที่ Dropdown เลือกวิชา ด้านบนสุด]**

---

ข้อมูลทั้ง 7 แท็บจะเปลี่ยนตามวิชาที่เลือกอัตโนมัติ

### 7.2 แท็บ 1 — ภาพรวม (Overview)

แสดง:
- กราฟแท่ง: อัตราผ่าน/ไม่ผ่านแต่ละโจทย์
- สถิติรวม: จำนวน Submission, อัตราผ่าน, คะแนนเฉลี่ย

---

📸 **[แคปเจอหน้าจอ: แท็บภาพรวม — แสดงกราฟ Bar Chart อัตราผ่านแต่ละโจทย์]**

---

### 7.3 แท็บ 2 — รายบุคคล

1. เลือกชื่อนักเรียนจาก Dropdown
2. ดูประวัติการส่งทุกครั้ง (คะแนน, วันที่, โค้ด)
3. กดปุ่ม **"สร้างรายงาน AI"** → Gemini วิเคราะห์จุดแข็ง/อ่อนของนักเรียนคนนั้น
4. กดปุ่ม **"วิเคราะห์ทุกคน (Bulk)"** → สร้างรายงาน AI สำหรับนักเรียนทุกคนพร้อมกัน

---

📸 **[แคปเจอหน้าจอ: แท็บรายบุคคล — แสดงตาราง Submission และ AI Report ด้านล่าง]**

---

### 7.4 แท็บ 3 — สรุปคะแนนทุกคน

ตารางแสดงคะแนนรวมของนักเรียนทุกคน สามารถ Sort ได้ตาม:
- เลขที่ (ค่าเริ่มต้น)
- คะแนนรวม
- ชื่อ

---

📸 **[แคปเจอหน้าจอ: แท็บสรุปคะแนน — แสดงตารางคะแนนทุกคน พร้อมปุ่ม Sort]**

---

### 7.5 แท็บ 4 — คะแนนฝึกเอง

แสดงข้อมูล Self-Practice ของนักเรียนแต่ละคน:
- จำนวนครั้งที่ฝึก
- คะแนนเฉลี่ย
- โจทย์ที่ฝึกบ่อย

---

### 7.6 แท็บ 5 — รายงาน AI

กดปุ่ม **"สร้างรายงานทั้งชั้น"** → Gemini วิเคราะห์ภาพรวมทั้งชั้นเรียน

รายงานครอบคลุม:
- โจทย์ที่นักเรียนส่วนใหญ่ทำไม่ผ่าน
- รูปแบบ Error ที่พบบ่อย
- ข้อแนะนำสำหรับการสอนครั้งถัดไป

---

📸 **[แคปเจอหน้าจอ: แท็บรายงาน AI — แสดง Report ที่ Gemini สร้าง พร้อมหัวข้อวิเคราะห์]**

## 8. Gamification Admin

กดเมนู **"Gamification"** ใน Navbar หรือกดปุ่ม **"Gamification Admin"** บน Dashboard

---

📸 **[แคปเจอหน้าจอ: หน้า Gamification Admin — แสดงแท็บทั้ง 4 ด้านบน]**

---

### 8.1 แท็บ Season

Season คือช่วงเวลาพิเศษที่นักเรียนได้รับ XP เพิ่มขึ้น (เช่น XP x2 ระหว่างสัปดาห์ก่อนสอบ)

**ดู Season ปัจจุบัน:**
- แสดงชื่อ Season, วันสิ้นสุด, และ XP Multiplier ที่ Active อยู่

**สร้าง Season ใหม่:**

| ช่อง | ตัวอย่าง |
|---|---|
| ชื่อ Season | Season 1 ภาคเรียน 1/2568 |
| วันสิ้นสุด | 30/04/2568 |
| XP Multiplier | 2 (= นักเรียนได้ XP สองเท่า) |

กด **"สร้าง Season"**

> Season ที่ Active จะแสดง Banner บนหน้า Leaderboard ของนักเรียน

---

📸 **[แคปเจอหน้าจอ: แท็บ Season — แสดง Form สร้าง Season ใหม่ และ Season ที่กำลัง Active]**

---

### 8.2 แท็บ Export ข้อมูล

กดปุ่ม **"📥 Export JSON"** → ดาวน์โหลดข้อมูล Gamification ทั้งหมดเป็นไฟล์ JSON

ข้อมูลที่ได้:
- `playerStats` (XP, Rank, Streak ทุกคน)
- `coachInteractions` (Log การใช้ AI Coach ล่าสุด 50 รายการ)

ใช้สำหรับวิเคราะห์ด้วย SPSS / R / Python

---

### 8.3 แท็บ Award XP

สำหรับให้ XP นักเรียนแบบ Manual (กรณีกิจกรรมพิเศษนอกระบบ)

1. เลือกนักเรียนจาก Dropdown
2. ระบุจำนวน XP
3. ระบุเหตุผล
4. กด **"มอบ XP"**

> XP ที่ Award จะถูกบันทึกใน `xpLedger` พร้อม source: 'manual'

---

📸 **[แคปเจอหน้าจอ: แท็บ Award XP — แสดง Form เลือกนักเรียนและช่องใส่จำนวน XP]**

---

### 8.4 แท็บ Game Stats

แสดงสถิติ Mini-Games รวม:
- จำนวน Session ต่อประเภทเกม
- คะแนนเฉลี่ย
- นักเรียนที่เล่นมากที่สุด

---

### 8.5 อัพเดท Leaderboard

ส่วน **"อัปเดต Leaderboard"** ด้านล่างหน้า:

1. เลือกรายวิชา (หรือปล่อยว่างเพื่ออัพเดทรวมทุกวิชา)
2. กด **"🔄 Refresh Leaderboard"**
3. ระบบอัพเดท Snapshot ทั้ง 3 ช่วงเวลา (Daily / Weekly / Alltime)

---

📸 **[แคปเจอหน้าจอ: ส่วน Refresh Leaderboard — แสดง Dropdown เลือกวิชา และปุ่ม Refresh]**

สร้าง Assignment + Test Cases แบบรวดเร็วในหน้าเดียว เหมาะสำหรับออกโจทย์จำนวนมากอย่างรวดเร็ว

### 9.2 Realtime Dashboard (`#/teacher/realtime`)

Monitor การส่งงานของนักเรียนแบบ Real-time เห็นสถานะทุกคนในเวลาจริง — ใช้ขณะนักเรียนทำข้อสอบ

### 9.3 Free Editor (`#/teacher/editor`)

Code Editor อิสระที่ไม่ผูกกับโจทย์ใดๆ:
- รองรับหลาย Theme (Dracula, Monokai, One Dark, Nord ฯลฯ)
- เลือก Font ได้ (JetBrains Mono, Fira Code ฯลฯ)
- มีฟีเจอร์ Drawing/Annotation สำหรับอธิบายโค้ด
- เหมาะสำหรับสาธิตหน้าชั้นเรียน

## 10. การส่งออกข้อมูลสำหรับวิจัย

### 10.1 Export CSV (สำหรับ SPSS/R)

เข้าระบบด้วย Admin Account → ไปที่ `#/admin/seed`

กดปุ่ม **"📥 Export CSV (SPSS/R)"**

ได้ไฟล์ CSV ประกอบด้วย:
```
uid, displayName, engagementTier, xp, streakDays,
totalGameSessions, quizBlitzSessions, codeAutopsySessions,
bugHuntSessions, selfPracticeCount, selfPracticeAvgScore,
e1ScoreUnit1, e1ScoreUnit2, e1ScoreUnit3, e1ScoreUnit4, e1Average
```

### 10.2 Export JSON (จาก Gamification Admin)

ไปที่ `#/teacher/gamification` → แท็บ **"Export ข้อมูล"** → กด **"📥 Export JSON"**

### 10.3 ข้อมูลที่เก็บอัตโนมัติสำหรับวิจัย

| Collection | ข้อมูล | เก็บเมื่อ |
|---|---|---|
| `xpLedger` | ทุก XP event | ส่งงาน, เล่นเกม, Streak |
| `coachInteractions` | Log AI Coach | ขอ Hint, ล้มเหลว 3 ครั้ง |
| `miniGameSessions` | ผลการเล่นเกม | จบแต่ละ Session |
| `studentAchievements` | Badge ที่ได้รับ | Unlock Achievement |
| `submissions` | ทุกการส่งงาน | กด Submit |
| `playerStats` | XP/Rank/Streak | อัพเดทหลังทุกกิจกรรม |

### C. คำถามที่พบบ่อย

**Q: นักเรียนลงทะเบียนเองได้มั้ย?**
A: ได้ ใช้ Class Code ของวิชา แต่ต้องรอครูอนุมัติก่อนเข้าเรียน

**Q: Leaderboard แสดงนักเรียนจากวิชาอื่นมั้ย?**
A: ไม่ แสดงเฉพาะนักเรียนในวิชาเดียวกัน

**Q: ถ้านักเรียนส่งงานผิดหลายครั้ง XP จะหักมั้ย?**
A: ไม่หัก แต่ได้ XP น้อยกว่า (Score ต่ำ = XP น้อย)

**Q: ครูสร้างวิชาใหม่ภาคเรียนใหม่ต้องทำอะไรบ้าง?**
A: สร้างวิชาใหม่ → ส่ง Class Code ให้นักเรียน → นักเรียนลงทะเบียน → Leaderboard ใหม่จะแยกอัตโนมัติ
